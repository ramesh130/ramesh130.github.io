Project OS Abstraction Layer

Platform Windows Mobile, Symbian, Linux Mobile

Client Symphony

There are a variety of Operating Systems currently available for Mobile Phones like Symbian, Windows Mobile, and Linux Mobile etc. Each of these Operating Systems command a fair share of the Mobile Phone market and many vendors provide Mobile Phones on different Operating Systems. Each of these Operating Systems provides a different programming model. The code written for one Operating System cannot be reused on other. Companies developing Mobile Application face a dilemma. If they restrict the application to single Operating System, they restrict the market of their application. Developing an application for a broad range of Mobile Phones entails writing the application for each of the Operating System. This translates to huge investments as the company has to maintain multiple teams. There is a duplication of effort, managing code base becomes difficult, making changes, adding new features is cumbersome and a substantial effort in testing has to be put in.

Using an Operating System Abstraction Layer can make the life much simpler for Companies involved in Mobile Application development. The Abstraction Layer provides an intuitive and easy to use interface for all of the Operating System tasks. It is independent of any Operating System and is self contained.

Project QwiKam

Platform .NET, Windows 2000/XP/Mobile, SQL, Brew, Symbian, J2ME

Client Sindhara SuperMedia

QwiKam is a powerful encoder/decoder for camera phones. Cell phones equipped with multi-million pixel cameras are not capable of displaying or transmitting images at resolution levels higher than 320x240. The inferior quality of encoded and transmitted images is disappointing to camera phone users. The JPEG encoder in the handset outputs encoded images with large file sizes, consumes too much battery power, and is therefore not useful for capturing and transmitting higher resolution images.

 

QwiKam is the ideal solution for encoding/decoding high resolution images in handsets, up to 1080x800 resolution. The 1080x800 image results in a file size of ~ 30 KB, as compared to the JPEG file size of >500 KB. Users can upload and download crisp, high quality, images to/from cell phones, desktops and web servers. The footprint of the QwiKam software codec is ~ 150 KB, and can be easily installed in all 2.5G/3G handsets.

 

Peer-to-peer high resolution picture messaging is made possible with QwiKam.

 

The QwiKam software codec can be downloaded or pre-installed into cell phones in BREW, Symbian, J2ME, and Smartphone (Windows Mobile) formats. APIs are also available for embedding the QwiKam codec into handsets.

 

Project QwikStream

Platform Windows 2000/XP/Mobile, Brew, Symbian, J2ME, PPC,

QwikStream represents the state-of-the-art in video compression making it possible to deliver high definition content in real time over standard internet, WiFi, or Satellite networks, and DVD quality video over cellular (GPRS, CDMA, Edge).  Built on Sindharas patented technology, video is transformed using a proprietary implementation of wavelet algorithms, compacted spatially using an Advanced Rapid Image Encoding Scheme (ARIES), and compacted temporally with a High Speed Motion Prediction (HSMP) routine.   This results in high quality, artifact free video that performs well in very constrained bandwidth environments. 

The QwikStream family consists of:

  • QwikStream - Designed for mobile devices, this version preserves battery life, renders best on mobile displays, streams over 2.5G & 3G networks, and is optimized for the processors common in PDAs and cell phones.

  • QwikStreamHD - Streams High Definition video (1080x700 @ 30 fps) for broadband environments.  Currently supported on Windows based machines.
  • QwikLive All versions of QwikStream can be enabled with QwikLive.  QwikLive allows real time video capture using most 3rd party cameras and instantaneous streaming.  Video created with QwikLive can be stored for later viewing or streamed over a network to be viewed immediately using any compatible viewer.

Project QwikTex
